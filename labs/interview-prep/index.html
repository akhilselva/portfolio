JavaScript Core Fundamentals 30 Qs

1. What is JavaScript execution context?

An execution context is the environment in which JS code runs. It contains the variable environment, lexical environment, scope chain, and this binding. There is a global execution context, function execution contexts, and module contexts.

2. What is the call stack?

The call stack tracks currently executing function calls. When a function is called, a frame is pushed. When it returns, the frame is popped. Synchronous JS runs top to bottom through this stack.

3. What is scope?

Scope defines where variables and functions are accessible. JavaScript has global scope, module scope, function scope, block scope, and lexical scope.

4. What is lexical scope?

Lexical scope means variable access is determined by where functions are written in source code, not where they are called.

5. What is a closure?

A closure is a function bundled with references to its surrounding lexical environment. It lets a function access variables from an outer scope even after that outer function has returned.

6. Why are closures useful?

Closures enable private state, function factories, memoization, callbacks that remember context, and module-like patterns.

7. What is hoisting?

Hoisting is the behavior where declarations appear available before their written position. Function declarations are initialized early. var declarations are hoisted but initialized as undefined. let and const are hoisted but unavailable before initialization due to the temporal dead zone.

8. Difference between var, let, and const?

var is function-scoped and can be redeclared. let and const are block-scoped. let can be reassigned. const cannot be reassigned, but object contents can still mutate.

9. What is the temporal dead zone?

The temporal dead zone is the period between entering a scope and the actual initialization of a let or const variable. Accessing it during this period throws a ReferenceError.

10. What is shadowing?

Shadowing happens when an inner scope declares a variable with the same name as an outer scope variable, hiding the outer one inside that block or function.

11. What is this in JavaScript?

this is a dynamic binding that usually depends on how a function is called. In object method calls, it points to the object. In standalone strict-mode functions, it is undefined. Arrow functions do not bind their own this.

12. Difference between regular function and arrow function?

Regular functions have their own this, arguments, and can be constructors. Arrow functions capture this lexically, do not have their own arguments, and cannot be used with new.

13. What are primitive values?

Primitive values are string, number, bigint, boolean, undefined, symbol, and null. They are immutable values, though variables holding them can be reassigned.

14. What are reference values?

Objects, arrays, functions, maps, sets, dates, and similar structures are reference values. Variables hold references to them, so mutation through one reference is visible through another.

15. What is pass by value vs pass by reference in JS?

JavaScript passes arguments by value. For objects, the value being passed is a reference. That means you can mutate the object, but reassigning the parameter does not reassign the caller variable.

16. What is type coercion?

Type coercion is automatic conversion between types, like string to number or boolean to number. It happens with operators like ==, +, -, if conditions, and comparisons.

17. Difference between == and ===?

=== checks equality without type coercion. == allows coercion. In interviews and production code, prefer === unless you intentionally want coercive behavior.

18. Why is typeof null object?

It is a long-standing JavaScript legacy bug. null is a primitive value, but typeof null returns object for historical compatibility.

19. What is NaN?

NaN means Not-a-Number. It is a number value representing invalid numeric results. NaN is not equal to itself, so use Number.isNaN to check it.

20. Difference between null and undefined?

undefined usually means a variable or property has not been assigned. null is an intentional empty value.

21. What is destructuring?

Destructuring extracts values from arrays or properties from objects into variables using pattern syntax.

22. What is spread syntax?

Spread expands iterable values or object properties into another array, argument list, or object. It is shallow, not deep.

23. What is rest syntax?

Rest collects remaining parameters, array elements, or object properties into a single variable.

24. What are template literals?

Template literals use backticks and allow embedded expressions, multi-line strings, and tagged templates.

25. What is optional chaining?

Optional chaining allows safe nested access using ?. If the left side is null or undefined, it returns undefined instead of throwing.

26. What is nullish coalescing?

?? returns the right-side value only when the left side is null or undefined. Unlike ||, it does not treat 0, false, or empty string as missing.

27. What are truthy and falsy values?

Falsy values are false, 0, -0, 0n, empty string, null, undefined, and NaN. Everything else is truthy.

28. What is the difference between map, filter, reduce, and forEach?

map transforms each item and returns a new array. filter returns matching items. reduce accumulates one result. forEach performs side effects and returns undefined.

29. When should you avoid reduce?

Avoid reduce when it makes code less readable. It is powerful for accumulation, grouping, flattening, and building maps, but simple loops are often clearer.

30. What is immutability?

Immutability means not changing existing data structures directly. Instead, create new versions. This helps predictable state updates, change detection, and debugging.

JavaScript Async, Promises, and Event Loop 26 Qs

1. What is the event loop?

The event loop coordinates execution between the call stack, task queues, microtask queues, timers, I/O callbacks, and rendering. It lets JavaScript handle async work without blocking the main execution thread.

2. What is a task/macrotask?

A task is work scheduled for a future event-loop turn, such as setTimeout callbacks, user events, script execution, and some I/O callbacks.

3. What is a microtask?

A microtask is high-priority async work that runs after the current stack clears and before the next task. Promise callbacks and queueMicrotask callbacks are common examples.

4. Promise.then vs setTimeout order?

Promise.then callbacks run as microtasks after the current synchronous code completes. setTimeout callbacks run as tasks in a later event-loop turn. So Promise.then usually runs before setTimeout 0.

5. What is a Promise?

A Promise represents eventual completion or failure of an asynchronous operation. It can be pending, fulfilled, or rejected.

6. What does async function return?

An async function always returns a Promise. Returning a value resolves the Promise. Throwing an error rejects it.

7. What does await do?

await pauses execution of the surrounding async function until the awaited Promise settles, then resumes with the fulfilled value or throws the rejection reason.

8. Does await block the whole thread?

No. await pauses only the async function. The event loop can continue processing other work.

9. What is Promise.all?

Promise.all runs promises concurrently and resolves when all fulfill. It rejects immediately when any promise rejects.

10. What is Promise.allSettled?

Promise.allSettled waits for every promise to settle and returns status objects for both fulfilled and rejected results.

11. What is Promise.race?

Promise.race settles with the first promise that fulfills or rejects.

12. What is Promise.any?

Promise.any fulfills with the first fulfilled promise. It rejects only if all promises reject.

13. Why can async code still be slow?

Async does not mean parallel CPU execution. If you await sequentially inside a loop, you may serialize operations. Use Promise.all for independent I/O tasks.

14. How do you handle errors in async/await?

Use try/catch around awaited calls, or attach .catch to promises. For concurrent operations, decide whether one failure should fail all or be isolated.

15. What is an unhandled promise rejection?

It happens when a Promise rejects and no rejection handler is attached. In production Node apps, this should be logged and treated seriously.

16. What is callback hell?

Deeply nested callbacks that make control flow hard to read and error handling difficult. Promises and async/await help flatten this structure.

17. What is inversion of control in callbacks?

When you pass a callback to another API, you rely on that API to call it correctly, once, at the right time, with the right arguments.

18. What is starvation in event loop?

Starvation happens when high-priority queues keep getting filled and prevent lower-priority work from running. In Node, excessive process.nextTick can starve I/O.

19. Browser event loop vs Node event loop?

Browsers coordinate JS tasks, microtasks, and rendering. Node coordinates timers, pending callbacks, poll, check, close callbacks, nextTick, and Promise microtasks. Similar idea, different runtime phases.

20. process.nextTick vs Promise microtask in Node?

process.nextTick callbacks run before Promise microtasks in Node, so abusing nextTick can delay Promise callbacks and I/O.

21. setImmediate vs setTimeout 0 in Node?

At top level, ordering can vary. Inside an I/O callback, setImmediate generally runs before setTimeout 0 because it runs in the check phase after poll.

22. Why did my promise log before timeout?

Because Promise callbacks are microtasks, and the event loop drains microtasks after the current synchronous code before moving to the next task such as a timer.

23. What is queueMicrotask?

queueMicrotask explicitly schedules a microtask. It is useful when you need code to run after current synchronous execution but before the next task.

24. What is debounce?

Debounce delays execution until activity stops for a specified duration. Common for search boxes and resize handlers.

25. What is throttle?

Throttle limits execution to at most once per interval. Common for scroll handlers and rate-limited UI actions.

26. What is AbortController?

AbortController provides a signal to cancel fetch requests or other abortable async operations.

Objects, Prototypes, Classes, and Memory 15 Qs

1. What is a prototype?

A prototype is an object from which another object can inherit properties and methods. JavaScript uses prototype chains for inheritance.

2. What is the prototype chain?

When accessing a property, JS first checks the object itself, then its prototype, and continues upward until null.

3. Class in JS vs classical OOP?

JavaScript class syntax is mostly syntactic sugar over prototypes. It provides cleaner syntax for constructors, methods, inheritance, and private fields.

4. What is new doing?

new creates a fresh object, links it to the constructor prototype, calls the constructor with this bound to the new object, and returns the object unless the constructor explicitly returns another object.

5. What is Object.create?

Object.create creates a new object with the given prototype. It is useful for explicit prototypal inheritance.

6. What is hasOwnProperty?

It checks whether a property exists directly on an object, not through the prototype chain. Modern code can use Object.hasOwn.

7. What is Object.freeze?

Object.freeze prevents adding, deleting, or changing own properties. It is shallow; nested objects can still mutate unless also frozen.

8. What is shallow copy?

A shallow copy copies top-level properties but nested objects remain shared references.

9. What is deep copy?

A deep copy recursively copies nested data. structuredClone handles many built-in types, but functions, DOM nodes, and some custom cases need care.

10. What is garbage collection?

Garbage collection automatically frees memory that is no longer reachable from roots like global objects and active stack frames.

11. What causes memory leaks in JS apps?

Common causes: forgotten timers, event listeners not removed, global references, large caches, closures retaining large objects, detached DOM nodes, and subscriptions not cleaned up.

12. How would you debug memory leaks?

Use browser/Node heap snapshots, allocation timelines, object retainers, listener audits, and reproduce memory growth with controlled steps.

13. Map vs Object?

Map supports any key type, preserves insertion order, has size, and is better for dynamic key-value storage. Object is better for structured records and JSON-like data.

14. Set vs Array?

Set stores unique values and offers fast membership checks. Array keeps order and duplicates and is better for indexed sequences.

15. WeakMap vs Map?

WeakMap keys must be objects and do not prevent garbage collection. Useful for metadata attached to objects without causing leaks.

Browser, DOM, HTTP, and Web Platform 25 Qs

1. What is the DOM?

The DOM is the browser representation of an HTML document as a tree of nodes that JavaScript can read and modify.

2. What is event bubbling?

Event bubbling means an event starts at the target element and propagates upward through ancestors.

3. What is event capturing?

Capturing is the phase where the event travels from the root down toward the target before bubbling back up.

4. What is event delegation?

Event delegation attaches one listener to a parent and handles events from children using bubbling. It reduces listeners and works with dynamic children.

5. What is preventDefault?

preventDefault stops the browser default action for an event, such as submitting a form or following a link.

6. What is stopPropagation?

stopPropagation prevents the event from continuing through the capture/bubble path.

7. What is reflow/layout?

Reflow is when the browser calculates element sizes and positions. It can be expensive if triggered repeatedly.

8. What is repaint?

Repaint updates pixels without necessarily recalculating layout. Color changes often cause repaint but not layout.

9. How do you reduce layout thrashing?

Batch DOM reads and writes, avoid repeated forced synchronous layout, use transforms for animations, and minimize direct DOM manipulation.

10. What is localStorage?

localStorage is persistent key-value storage in the browser. It is synchronous and string-only, so avoid large or sensitive data.

11. What is sessionStorage?

sessionStorage is similar to localStorage but scoped to a browser tab/session.

12. What are cookies?

Cookies are small key-value data sent with HTTP requests to matching domains. They are often used for sessions and can be configured with HttpOnly, Secure, SameSite, path, and expiry.

13. localStorage vs cookie for auth token?

HttpOnly Secure SameSite cookies are generally safer against XSS token theft. localStorage is easy but vulnerable if XSS occurs.

14. What is CORS?

CORS is a browser security mechanism controlling which origins can read responses from another origin. It uses headers like Access-Control-Allow-Origin and preflight OPTIONS requests.

15. What is same-origin policy?

Same-origin policy restricts scripts from one origin from reading data from another origin unless allowed by mechanisms like CORS.

16. What is a preflight request?

A browser sends an OPTIONS request before certain cross-origin requests to check if the actual method and headers are allowed.

17. What is caching?

Caching stores responses or computed data closer to where they are needed, reducing latency, bandwidth, and origin load.

18. What is Cache-Control?

Cache-Control is an HTTP header that tells browsers and intermediaries how to cache a response, using directives like max-age, no-store, no-cache, private, and public.

19. What is ETag?

ETag is a validator for cached content. Clients can send If-None-Match, and servers can return 304 Not Modified if content has not changed.

20. GET vs POST?

GET is for safe retrieval and can be cached/bookmarked. POST usually creates or triggers processing and has a request body. Do not use GET for state-changing operations.

21. PUT vs PATCH?

PUT usually replaces a resource representation. PATCH partially updates it.

22. What is idempotency?

An operation is idempotent if repeating it produces the same final state. GET, PUT, and DELETE should be idempotent; POST usually is not unless designed with idempotency keys.

23. What are status code categories?

2xx success, 3xx redirection, 4xx client errors, 5xx server errors.

24. 401 vs 403?

401 means unauthenticated or invalid credentials. 403 means authenticated but not authorized.

25. What is HTTPS?

HTTPS is HTTP over TLS, providing encryption, integrity, and server authentication.

Vue 3 Fundamentals and Reactivity 27 Qs

1. What is Vue reactivity?

Vue reactivity tracks state access and mutations so the UI can update when reactive state changes.

2. How does Vue 3 reactivity work?

Vue 3 uses JavaScript Proxies to intercept property reads and writes. Reads are tracked as dependencies; writes trigger effects and component updates.

3. ref vs reactive?

ref wraps a value in an object with .value and works for primitives or objects. reactive returns a Proxy for objects. In templates, refs are usually unwrapped automatically.

4. When should you use ref?

Use ref for primitives, simple independent state, DOM refs, and values that may be replaced entirely.

5. When should you use reactive?

Use reactive for grouped object state where properties belong together and you prefer object-style access.

6. What is computed?

computed creates a cached derived value that updates only when its reactive dependencies change.

7. computed vs method?

computed is cached based on dependencies. A method runs every time it is called or render uses it.

8. watch vs watchEffect?

watch tracks explicit sources and gives old/new values. watchEffect automatically tracks whatever reactive values are accessed during its execution.

9. When is watch useful?

Use watch for side effects caused by state changes: API calls, persistence, analytics, debounced search, or reacting to route changes.

10. What is nextTick in Vue?

nextTick waits until Vue has flushed pending DOM updates after reactive state changes.

11. Why does DOM not update immediately after state change?

Vue batches updates for performance. State changes schedule a render update, and DOM changes happen in the next flush.

12. What is component lifecycle?

Lifecycle hooks let code run at stages like setup, mounted, updated, and unmounted.

13. What should be cleaned up on unmounted?

Timers, event listeners, subscriptions, sockets, observers, and pending async work where relevant.

14. What is props down, events up?

Parent passes data to child via props. Child communicates changes/events upward using emits.

15. Why should props not be mutated directly?

Props belong to the parent. Mutating them in a child breaks one-way data flow and can be overwritten by parent updates.

16. What is v-model?

v-model is syntax sugar for passing a value prop and listening to an update event, commonly modelValue and update:modelValue in Vue 3.

17. What is a slot?

A slot lets a parent pass template content into a child component.

18. What are scoped slots?

Scoped slots let a child expose data to slot content provided by the parent.

19. What is provide/inject?

provide/inject passes dependencies down the component tree without prop drilling. Use carefully to avoid hidden coupling.

20. What is Pinia?

Pinia is Vue’s recommended state management library. It provides stores with state, getters, actions, devtools support, and modular structure.

21. Pinia getters vs actions?

Getters derive state like computed values. Actions contain business logic and can be async.

22. When should state be local vs store?

Keep state local if only one component or small subtree needs it. Move to store when multiple distant components need shared state or business operations.

23. What is hydration?

Hydration connects client-side JS to server-rendered HTML so the app becomes interactive.

24. What causes hydration mismatch?

Different server/client output due to random values, dates, browser-only APIs, async data differences, or conditional rendering that differs between server and client.

25. How do you optimize Vue performance?

Use computed carefully, avoid unnecessary reactive deep objects, split components, use v-memo/v-once where suitable, virtualize long lists, lazy-load routes, and avoid expensive work in render.

26. What is key in v-for?

key gives Vue stable identity for list items. It helps efficient DOM updates and prevents state bugs when items reorder.

27. Why avoid index as key?

Index keys break identity when list order changes, causing incorrect component state reuse.

Node.js and Express Fundamentals 24 Qs

1. What is Node.js?

Node.js is a JavaScript runtime built on V8 that runs outside the browser and provides APIs for files, networking, processes, streams, and servers.

2. Why is Node good for I/O-heavy apps?

Node uses non-blocking I/O and an event loop, allowing many concurrent connections without one thread per request.

3. Where is Node weak?

CPU-heavy work can block the event loop unless offloaded to worker threads, child processes, native services, or separate compute systems.

4. What is libuv?

libuv is the library behind Node’s event loop, async I/O, thread pool, timers, and cross-platform abstractions.

5. What uses the Node thread pool?

Some file system operations, DNS lookup variants, crypto, compression, and other expensive async operations can use libuv’s thread pool.

6. What is event-loop blocking?

Long synchronous CPU work prevents Node from handling other callbacks, causing latency spikes for all requests in that process.

7. How do you detect event-loop blocking?

Measure event loop lag, p95/p99 latency, CPU usage, flamegraphs, async hooks/profilers, and request timing.

8. Common ways to avoid blocking Node?

Use async APIs, stream large data, paginate, offload CPU work, avoid huge JSON parsing/stringifying in hot paths, and avoid sync filesystem calls in request handlers.

9. What is a stream?

A stream processes data in chunks instead of loading everything into memory. Useful for files, uploads, downloads, compression, and network data.

10. What is backpressure?

Backpressure happens when the consumer cannot process data as fast as the producer sends it. Streams signal this so producers slow down.

11. CommonJS vs ES Modules?

CommonJS uses require/module.exports and loads synchronously. ES Modules use import/export, static analysis, and support top-level await.

12. What is middleware in Express?

Middleware is a function that receives req, res, and next. It can modify request/response, end the response, or pass control to the next middleware.

13. What order does Express middleware run?

Middleware runs in the order it is registered. Route-specific middleware runs when its path/method matches.

14. How do you handle errors in Express?

Use error-handling middleware with four arguments: err, req, res, next. In async routes, ensure rejected promises are passed to error handling depending on Express version/pattern.

15. What is req.params vs req.query vs req.body?

req.params are path variables. req.query is URL query string data. req.body is parsed request body data from middleware such as express.json.

16. What is helmet?

Helmet sets security-related HTTP headers to reduce common web vulnerabilities.

17. Why validate input?

All external input is untrusted. Validation prevents malformed data, injection, crashes, authorization bypasses, and bad data entering the system.

18. What is rate limiting?

Rate limiting restricts how often a client can perform actions, protecting against abuse, brute force, scraping, and accidental overload.

19. What is graceful shutdown?

Graceful shutdown stops accepting new requests, lets in-flight requests finish within a timeout, closes connections/resources, and exits cleanly.

20. What should production logging include?

Request IDs, timestamps, route, status, latency, user/client context where safe, errors with stack traces, and no sensitive secrets.

21. What is correlation ID?

A request ID propagated across services so logs and traces for one user action can be connected.

22. What is clustering in Node?

Running multiple Node processes to use multiple CPU cores. Modern deployments often use container replicas or process managers instead.

23. What is worker_threads?

worker_threads allow CPU-heavy JS to run in separate threads without blocking the main event loop.

24. How do you secure an Express API?

Use TLS, secure headers, input validation, authentication, authorization, safe cookies, CORS restrictions, rate limiting, dependency updates, logging, and error hygiene.

Cloudflare Workers, Serverless, and Deployment 17 Qs

1. What is a Cloudflare Worker?

A Worker is serverless JavaScript/TypeScript code running on Cloudflare’s edge/runtime. It is good for lightweight request handling, APIs, auth gates, redirects, caching, and transformations.

2. Worker vs traditional server?

Worker is stateless, auto-scaled, and edge-distributed with runtime limits. A server gives more control, long-running processes, broader runtime support, and better fit for heavy compute/background work.

3. What are Workers bad at?

Long-running CPU-heavy jobs, full OS-level dependencies, native binaries, heavy browser automation, large file processing, and workflows needing persistent in-process state.

4. What are environment bindings?

Bindings connect Workers to services like D1, KV, R2, Durable Objects, secrets, service bindings, and rate-limiters through env.

5. What is KV?

Workers KV is a globally distributed key-value store suited for read-heavy, eventually consistent data.

6. What is D1?

D1 is Cloudflare’s serverless SQL database based on SQLite, useful for small to medium relational app data near Workers.

7. What is R2?

R2 is Cloudflare object storage, useful for files, images, exports, and large blobs.

8. What are Durable Objects?

Durable Objects provide strongly consistent per-object coordination and state. Useful for counters, sessions, collaborative rooms, rate limiters, and ordered workflows.

9. How would you deploy Worker via GitHub Actions?

Install dependencies, run tests/lint, authenticate using Cloudflare API token, run wrangler deploy, and separate preview/production environments by branch.

10. What is wrangler.toml/json?

Wrangler config defines Worker name, entry point, compatibility date, routes, bindings, vars, and environment-specific settings.

11. What is compatibility date?

A Cloudflare Workers setting that locks runtime behavior to a date so platform changes do not unexpectedly break your Worker.

12. What is a route in Cloudflare Workers?

A route maps a domain/path pattern to a Worker, such as wishlist-api.akhilselva.com/*.

13. How to serve Worker on custom subdomain?

Create a proxied DNS record for the subdomain, then attach the Worker route to subdomain/*.

14. Why might workers.dev show 404?

The Worker or Pages project may deploy successfully but not serve the expected path/file. For Pages, the HTML entry file and build output path must match what Cloudflare serves.

15. How do you handle secrets?

Use wrangler secret or Cloudflare dashboard secrets. Do not commit secrets into GitHub.

16. How do you avoid local config overwriting dashboard changes?

Treat code/config as source of truth. Pull dashboard changes into local wrangler config before deploying, or avoid manual dashboard edits for deploy-critical settings.

17. What is fail-open vs fail-closed for limits/security?

Fail-open bypasses the Worker if limits are hit. Fail-closed returns an error. Use fail-closed for security-critical routes.

API Design and Security 24 Qs

1. What makes a good REST API?

Clear resource-oriented URLs, correct HTTP methods/status codes, validation, pagination, filtering, idempotency, versioning strategy, auth, consistent errors, and observability.

2. How do you design error responses?

Return consistent shape: code, message, details, requestId. Avoid leaking stack traces or secrets to clients.

3. What is pagination?

Pagination limits response size. Offset pagination is simple. Cursor pagination is better for large changing datasets.

4. Offset vs cursor pagination?

Offset uses page/limit but gets slow and unstable on large changing data. Cursor uses a stable sort key and is better for infinite scroll and large datasets.

5. What is filtering?

Filtering lets clients narrow resources using query parameters. Validate allowed fields/operators to avoid expensive or unsafe queries.

6. What is sorting?

Sorting orders results. Only allow indexed/safe sort fields for large collections.

7. What is API versioning?

A strategy to evolve APIs without breaking clients. Common approaches: URL versioning, header versioning, or backward-compatible additive changes.

8. What is authentication?

Authentication verifies who the user/client is.

9. What is authorization?

Authorization verifies whether the authenticated identity can perform a specific action on a specific resource.

10. What is RBAC?

Role-Based Access Control grants permissions based on roles such as admin, editor, user.

11. What is ABAC?

Attribute-Based Access Control uses user, resource, action, and context attributes for decisions.

12. What is JWT?

JWT is a signed token format containing claims. It is self-contained, but revocation and storage need careful design.

13. Session cookie vs JWT?

Session cookie stores an opaque session ID and server-side session state. JWT stores claims client-side and is verified by signature. Cookies are often easier to revoke when backed by server sessions.

14. What is CSRF?

Cross-Site Request Forgery tricks a browser into sending authenticated requests using existing cookies. SameSite cookies and CSRF tokens help prevent it.

15. What is XSS?

Cross-Site Scripting injects malicious script into a page. Prevent with escaping, sanitization, CSP, safe frameworks, and avoiding dangerous HTML insertion.

16. What is SQL injection?

SQL injection happens when user input is concatenated into SQL. Prevent with parameterized queries and validation.

17. What is NoSQL injection?

NoSQL injection abuses query objects/operators, such as passing Mongo operators through unvalidated input. Prevent with schema validation and allowlisted query shapes.

18. What is mass assignment?

Mass assignment happens when clients can set fields they should not control, like role or isAdmin. Prevent using DTOs and allowlisted fields.

19. What is BOLA/IDOR?

Broken Object Level Authorization happens when users can access objects by changing IDs. Always check ownership/permission for each object access.

20. What is sensitive data exposure?

Returning or logging secrets, tokens, personal data, internal IDs, or fields not needed by the client. Use response shaping and least privilege.

21. How do you secure public search APIs?

Use rate limits, bot detection where needed, request tokens/nonces for abuse resistance, input constraints, caching, WAF rules, and monitoring.

22. What is idempotency key?

A client-provided unique key that lets the server safely deduplicate retries for operations like payments or order creation.

23. How do you design retries?

Retry only safe/idempotent operations or operations protected by idempotency keys. Use exponential backoff, jitter, max attempts, and clear timeout behavior.

24. What is circuit breaker?

A circuit breaker stops calling a failing dependency temporarily, preventing cascading failures and giving the dependency time to recover.

Databases, SQL, NoSQL, and Caching 27 Qs

1. What is an index?

An index is a data structure that speeds reads by avoiding full scans, at the cost of extra storage and slower writes.

2. When should you add an index?

Add indexes for frequent filters, joins, sorting, and uniqueness constraints. Use query plans to confirm benefit.

3. Why not index everything?

Indexes increase storage, memory pressure, and write cost because inserts/updates/deletes must update indexes too.

4. What is a primary key?

A primary key uniquely identifies each row in a table.

5. What is a foreign key?

A foreign key enforces relationship integrity between tables by referencing another table’s key.

6. What is normalization?

Normalization structures data to reduce duplication and anomalies. It usually splits data into related tables.

7. What is denormalization?

Denormalization duplicates or precomputes data to improve read performance, often at the cost of write complexity and consistency.

8. What is a transaction?

A transaction groups operations so they succeed or fail as one logical unit.

9. What is ACID?

Atomicity, Consistency, Isolation, Durability. These properties describe reliable transaction behavior.

10. What is isolation level?

Isolation level controls how concurrent transactions see each other’s changes.

11. What is Read Committed?

A transaction sees only committed data. In PostgreSQL it is the default isolation level.

12. What is Repeatable Read?

A transaction sees a stable snapshot across its duration, avoiding non-repeatable reads.

13. What is Serializable?

The strongest isolation level, behaving as if transactions ran one at a time, often requiring retries on serialization failures.

14. What is MVCC?

Multi-Version Concurrency Control keeps multiple row versions so readers and writers can work concurrently without many blocking locks.

15. What is a deadlock?

A deadlock occurs when transactions wait on each other in a cycle. DBs detect and abort one transaction.

16. SQL vs NoSQL?

SQL databases are relational, schema-driven, and strong for joins/transactions. NoSQL databases vary by model and often trade relational structure for flexible schema, scale, or access-pattern optimization.

17. MongoDB document design principle?

Model data based on access patterns. Embed when data is read together and bounded. Reference when data is large, shared, or independently updated.

18. What is MongoDB aggregation?

Aggregation processes documents through stages to filter, group, transform, sort, and compute results.

19. What is Redis used for?

Redis is commonly used for caching, rate limiting, queues, pub/sub, distributed locks, sessions, and fast counters.

20. What is cache-aside?

App checks cache first. On miss, app reads DB, stores result in cache, then returns. On writes, invalidate or update cache.

21. What is write-through cache?

Writes go through cache and DB together, keeping cache warm but adding write latency.

22. What is cache invalidation?

The process of removing/updating stale cache entries when underlying data changes. It is one of the hardest parts of caching.

23. What is TTL?

Time to live defines how long cached data remains valid before expiring.

24. What is cache stampede?

Many requests hit the DB at once after a popular cache key expires. Prevent with locks, stale-while-revalidate, jittered TTLs, and request coalescing.

25. What is Redis eviction?

When Redis reaches max memory, it removes keys based on configured eviction policy, such as LRU, LFU, random, or TTL-based policies.

26. What is read replica?

A database copy used for read traffic. It improves read scale but may lag behind the primary.

27. What is sharding?

Splitting data across multiple nodes by key to scale storage/write throughput. It increases operational and query complexity.

System Design and Production Thinking 30 Qs

1. How do you start a system design answer?

Clarify requirements, users, scale, read/write patterns, latency, consistency, data model, APIs, high-level architecture, bottlenecks, failure modes, and tradeoffs.

2. Functional vs non-functional requirements?

Functional requirements describe what the system does. Non-functional requirements describe quality constraints like latency, availability, scale, security, consistency, and cost.

3. What is scalability?

Ability of a system to handle growth in traffic, data, or complexity without unacceptable performance or cost.

4. Vertical vs horizontal scaling?

Vertical scaling adds power to one machine. Horizontal scaling adds more machines/instances.

5. What is load balancing?

Distributing traffic across multiple instances to improve availability and throughput.

6. What is stateless service?

A service that does not rely on local memory for user/session state. Any instance can handle any request.

7. Why prefer stateless APIs?

They are easier to scale horizontally, restart, deploy, and load balance.

8. What is sticky session?

A load balancer sends a user to the same backend instance. Useful sometimes, but reduces flexibility and can hide statefulness.

9. What is availability?

The percentage of time a system is operational and usable.

10. What is reliability?

The ability to perform correctly over time, including handling failures gracefully.

11. What is durability?

The guarantee that once data is written successfully, it will not be lost.

12. What is consistency?

Whether users/readers see the latest correct data across replicas and operations.

13. Strong vs eventual consistency?

Strong consistency means reads reflect latest committed writes. Eventual consistency means replicas converge over time but temporary stale reads are possible.

14. What is CAP theorem?

In a network partition, a distributed system must choose between consistency and availability. Partition tolerance is unavoidable in distributed systems.

15. What is queue-based architecture?

Producers put work into a queue. Workers process asynchronously. This smooths spikes, decouples services, and enables retries.

16. When use queues?

Use queues for email, notifications, uploads, batch jobs, retries, fan-out, and any work that does not need immediate synchronous completion.

17. What is at-least-once delivery?

Messages may be delivered more than once. Consumers must be idempotent.

18. What is exactly-once delivery?

A difficult guarantee. Many systems provide effectively-once behavior using idempotency, dedupe keys, and transactional processing.

19. What is backoff with jitter?

Retry delay increases over attempts and adds randomness to avoid synchronized retry storms.

20. What is graceful degradation?

System continues with reduced functionality when dependencies fail, such as showing cached results or hiding optional widgets.

21. What is fallback?

An alternate path when primary logic fails, like falling back from LLM search parsing to legacy search logic.

22. What is feature flag?

A runtime switch to enable, disable, or rollout features without redeploying.

23. What is A/B testing?

Comparing variants with real users to measure impact on metrics before full rollout.

24. What is observability?

Ability to understand system behavior using logs, metrics, traces, alerts, and dashboards.

25. What are SLO, SLA, SLI?

SLI is a measured indicator, SLO is target objective, SLA is contractual promise.

26. What are p95 and p99 latency?

Percentile latency values. p95 means 95 percent of requests are faster than that value. They reveal tail latency better than averages.

27. What is capacity planning?

Estimating resources needed for expected traffic, peak load, growth, and failure scenarios.

28. What is a hot partition?

A shard/partition receiving disproportionate traffic, causing bottlenecks despite overall capacity.

29. What is CDN?

A content delivery network caches and serves content closer to users, reducing latency and origin load.

30. What is edge computing?

Running code near users geographically, useful for routing, auth, caching, personalization, and lightweight APIs.

Frontend Engineering and Performance 18 Qs

1. What is SSR?

Server-Side Rendering generates HTML on the server for faster initial content and better SEO.

2. What is CSR?

Client-Side Rendering sends JS to the browser, which then builds the UI. It can be highly interactive but slower for first meaningful content if not optimized.

3. What is SSG?

Static Site Generation builds HTML at deploy time. Good for content pages that do not change per request.

4. SSR vs CSR for SEO?

SSR/SSG usually help search engines and social crawlers see content immediately, improving indexability for content-heavy pages.

5. What is lazy loading?

Loading code/images/components only when needed. It reduces initial bundle size and improves startup.

6. What is code splitting?

Splitting JS into smaller chunks loaded on demand, usually by route or feature.

7. What is tree shaking?

Build tools remove unused exports from bundles when modules are statically analyzable.

8. What is bundle size?

Total JS/CSS/assets sent to users. Larger bundles increase download, parse, and execution time.

9. What are Core Web Vitals?

Key UX performance metrics including LCP, INP, and CLS.

10. What is LCP?

Largest Contentful Paint measures when the largest visible content element is rendered.

11. What is CLS?

Cumulative Layout Shift measures unexpected visual movement during page load/lifetime.

12. What is INP?

Interaction to Next Paint measures responsiveness to user interactions.

13. How to optimize images?

Use correct dimensions, modern formats, compression, lazy loading, responsive srcset, CDN transformations, and preloading for critical images.

14. What is prefetch vs preload?

preload fetches critical resources needed soon for current page. prefetch fetches likely future resources with lower priority.

15. What is accessibility?

Making apps usable by people with disabilities. Includes semantic HTML, keyboard support, ARIA where needed, contrast, labels, focus management, and screen-reader compatibility.

16. Why semantic HTML?

Semantic elements provide built-in meaning, accessibility, SEO, and better browser behavior.

17. What is ARIA?

ARIA adds accessibility semantics when native HTML is insufficient. Prefer native semantic HTML first.

18. What is focus management?

Ensuring keyboard focus moves logically, especially in modals, menus, route changes, and dynamic UI.

Testing, Debugging, Git, and Code Quality 15 Qs

1. Unit vs integration vs e2e tests?

Unit tests verify small isolated logic. Integration tests verify multiple parts together. E2E tests simulate user flows across the app.

2. What should be unit tested?

Pure functions, validation, transformations, edge cases, reducers/stores, and business rules.

3. What should be integration tested?

API routes, DB interactions, component interactions, auth flows, and service boundaries.

4. What should be e2e tested?

Critical user journeys like login, checkout, search, booking, wishlist flows, and admin moderation.

5. What is mocking?

Replacing a dependency with a controlled fake to isolate the unit under test.

6. What is flaky test?

A test that passes/fails inconsistently without code changes. Usually due to timing, network, shared state, or non-determinism.

7. How do you debug production issue?

Reproduce if possible, check recent changes, logs, metrics, traces, error rates, affected users, roll back/mitigate, identify root cause, add prevention.

8. What is root cause analysis?

A structured investigation into why an incident happened, focusing on systems/process fixes rather than blame.

9. What makes code maintainable?

Clear naming, small functions, cohesive modules, tests, low coupling, explicit error handling, simple control flow, and documented tradeoffs.

10. What is technical debt?

A design/code shortcut that creates future cost. Not all debt is bad if intentional, visible, and repaid.

11. What should you check in code review?

Correctness, edge cases, readability, security, performance, tests, API contracts, error handling, observability, and maintainability.

12. What is Git rebase?

Rebase reapplies commits on top of another base, creating linear history. Avoid rebasing shared public commits unless team agrees.

13. Merge vs rebase?

Merge preserves branch history with merge commits. Rebase creates linear history by replaying commits.

14. What is CI/CD?

Continuous Integration runs checks on changes. Continuous Delivery/Deployment automates release to environments.

15. What should a pipeline include?

Install, lint, typecheck, unit tests, build, security/dependency checks, integration tests if feasible, artifact creation, and deploy with environment gates.

Interview Trap Questions and Crisp Answers 17 Qs

1. Console log order: sync, promise, timeout?

Synchronous logs run first. Promise callbacks run next as microtasks. setTimeout runs later as a task.

2. Why does process.nextTick run before Promise in Node?

Node has a special nextTick queue that is drained before the Promise microtask queue after the current operation.

3. Can const object be mutated?

Yes. const prevents reassignment of the variable binding, not mutation of the object it references.

4. Why does 0.1 + 0.2 not equal 0.3 exactly?

JavaScript numbers use binary floating-point. Some decimals cannot be represented exactly, causing tiny precision errors.

5. What does [] + [] return?

An empty string, because arrays convert to strings and empty arrays stringify to empty strings.

6. What does [] + {} return?

Usually "[object Object]" in expression context, because [] becomes empty string and object becomes string.

7. What does typeof NaN return?

number, because NaN is a special numeric value.

8. Why can a method lose this?

If you extract a method into a variable and call it standalone, the object call-site is lost. Bind it or use an arrow wrapper when needed.

9. Why is direct prop mutation bad in Vue?

The parent owns props. Child mutation breaks one-way data flow and may be overwritten on parent re-render.

10. Why is index key bad in v-for?

It causes identity bugs when list items are inserted, deleted, or reordered because component instances can be reused for the wrong item.

11. Why can await inside for-loop be bad?

It runs operations sequentially. For independent operations, use Promise.all with mapping and controlled concurrency if needed.

12. Why can JSON.stringify be dangerous for large objects?

It is synchronous and CPU/memory heavy. It can block the event loop and fail on circular references.

13. How to answer “what is reactivity” in 20 seconds?

Reactivity is dependency tracking between state and effects. In Vue, reading reactive state tracks the component/computed/watch as a dependency, and writing to that state triggers only the affected updates.

14. How to answer “what is event loop” in 20 seconds?

The event loop is the runtime scheduler. It runs synchronous code on the call stack, then drains microtasks like Promises, then moves to queued tasks like timers/I/O, allowing async behavior without blocking everything.

15. How to answer “Node is single-threaded?”

JavaScript execution in a Node process is mainly single-threaded, but Node uses libuv, OS async I/O, and a thread pool for certain operations. CPU-heavy JS can still block the event loop.

16. How to answer “microservices experience?”

I have worked on systems split into separate APIs/services with clear ownership and integration points. My strength is API-layer design, integration, contracts, fallback behavior, deployment awareness, and coordinating with platform/data teams.

17. How to answer “do you know Kubernetes?” honestly?

I have deployment exposure through GitLab CI/CD and platform-provided Kubernetes configs, but I was not the primary cluster operator. I understand how app deploys flow through containers/environments and can work with platform teams effectively.
